## What

Replace the default Tauri placeholder icons with a real gello mark, and wire
it through every platform target.

**The mark** (concept B, chosen for now): three kanban **columns** of equal
height as a blue ramp — `#4c8bf5`, `#7aa9f8`, `#31415a` (front→back
depth) — centered on a dark rounded-square tile (`#1b2430`). Reads as a
board and holds up at 32px. Provisional: the pipeline lets us swap the art
later without re-touching config.

**Production**: an agent-drafted **master SVG** is the single source; a
1024×1024 PNG rendered from it feeds `tauri icon`, which generates the full
platform set (macOS `.icns`, Windows `.ico`, Linux/tray PNGs, and the
Windows Store `Square*Logo.png` tiles) into `src-tauri/icons/`. The tile's
dark background and rounded corners are baked into the source (macOS does
not mask app icons), so the art is self-contained. The web favicon (Vite /
`index.html`, relevant once headless/web lands, c0069) is replaced with the
same mark.

## Discussion

- **Concept B, "for now"**: the columns-on-dark reads as kanban and survives
  small; picked provisionally over the `g` monograms (D–F) and the
  card+check (C). The columns→`g` fusion was explored and rejected as clumsy.
  The pipeline makes swapping the art cheap, so locking B now isn't binding.
- **Single SVG source → `tauri icon`**: one master, all sizes generated by
  the standard Tauri tool — no hand-maintained per-size PNGs. Re-theming is
  editing one file and re-running the command.
- **Tile baked into the source**: macOS doesn't round or background app
  icons (unlike iOS), so the dark rounded-square is part of the art, sized
  with the right padding/corner radius for the macOS look; the same square
  serves Windows/Linux.
- **Full set incl. favicon**: desktop icons, Store tiles (already present as
  `Square*Logo.png`), and the web favicon all come from the one mark, so the
  identity is consistent across the desktop app and the future web UI (c0069).
- **Agent-drafted art**: good enough for a dev tool and fully replaceable; a
  designer can supply a new master later and the pipeline is unchanged.
- **Open**: exact tile corner radius / padding to match the macOS squircle;
  whether to add a light-tile variant for contexts that sit on dark chrome;
  final ramp direction (which column is the accent) — cosmetic, settle when
  drawing the master.

## Notes

- Master: `src-tauri/app-icon.svg` — 1024×1024, dark tile (`#1b2430`, rounded
  rect rx 188 with ~100px transparent margin for the macOS icon grid), three
  equal columns in the ramp `#4c8bf5 / #7aa9f8 / #31415a`. `tauri icon` takes
  the SVG directly (rasterizes via resvg), so no hand-made PNG.
- `npx tauri icon src-tauri/app-icon.svg` regenerated all of `src-tauri/icons/`
  (32/64/128/128@2x png, icon.icns, icon.ico, Windows Store `Square*Logo.png`).
  Removed the iOS/Android output dirs it also created — out of scope for the
  desktop app.
- Favicon: `public/gello.svg` — a fuller-bleed variant (less padding, larger
  bars) so the columns hold at 16px; `index.html` `<link rel=icon>` repointed
  from the dead `/vite.svg` to `/gello.svg`.
- Verified: master + generated 128px raster render crisp (columns, ramp,
  padding correct); favicon serves 200 and renders at ~50px in the dev server.
  Gates: 426 tests green, typecheck + lint clean. The live OS dock/taskbar
  icon is confirmed on the next `tauri dev`/build.
- Provisional per the discussion — re-theming is editing `app-icon.svg` +
  re-running `tauri icon`.
